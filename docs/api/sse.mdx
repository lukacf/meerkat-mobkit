MobKit uses Server-Sent Events (SSE) for real-time communication between the console and the runtime. The SSE endpoint enables sending messages to agents and receiving streamed responses.

## Endpoint

```
POST /interactions/stream
```

## Request

Send a JSON body with the target member and message:

```json
{
  "member_id": "lead-1",
  "message": "Analyze the latest deployment metrics"
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `member_id` | `string` | yes | Target mob member ID |
| `message` | `string` | yes | Message text to send |

## Response

The response is an SSE stream (`Content-Type: text/event-stream`). Events are sent as the agent processes the message.

### Frame format

```
id: frame-001
event: agent_event
data: {"type": "text_delta", "delta": "Analyzing the metrics now..."}

id: frame-002
event: agent_event
data: {"type": "tool_call", "name": "query_metrics", "args": {...}}

id: frame-003
event: agent_event
data: {"type": "text_delta", "delta": "The deployment shows a 12% latency increase."}

id: frame-004
event: done
data: {"type": "turn_complete"}
```

Each frame has:

| Field | Description |
|-------|-------------|
| `id` | Unique frame identifier |
| `event` | Event type: `agent_event`, `module_event`, `done`, `error` |
| `data` | JSON payload specific to the event type |

### Event types

| Event | Description |
|-------|-------------|
| `agent_event` | Agent response chunk (text delta, tool call, reasoning) |
| `module_event` | Module-level event (health change, routing) |
| `done` | Stream complete |
| `error` | Error occurred during processing |

## Keep-alive

The server sends keep-alive frames at a configurable interval (default: 15 seconds) to prevent connection timeouts:

```
: keep-alive
```

The keep-alive interval can be configured when constructing the SSE router:

```rust
interaction_sse_router_with_keep_alive_interval(
    runtime,
    Duration::from_secs(30),
)
```

## Client-side parsing

The console client uses `parseSseFrames()` to extract structured frames from the raw SSE text:

```typescript
import { parseSseFrames } from "./lib/network";

const response = await fetch("/interactions/stream", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ member_id: "lead-1", message: "Status?" }),
});

const reader = response.body.getReader();
const decoder = new TextDecoder();

while (true) {
  const { done, value } = await reader.read();
  if (done) break;

  const frames = parseSseFrames(decoder.decode(value));
  for (const frame of frames) {
    switch (frame.event) {
      case "agent_event":
        handleAgentEvent(frame.data);
        break;
      case "done":
        handleComplete();
        break;
    }
  }
}
```

## Error handling

If the target member does not exist or the message cannot be delivered, the stream returns an error frame:

```
event: error
data: {"type": "member_not_found", "message": "No member with ID 'invalid-id'"}
```

Common error types:

| Error type | Cause |
|------------|-------|
| `member_not_found` | Target member ID does not exist |
| `session_error` | Meerkat session error |
| `mob_error` | Mob runtime error |
| `send_error` | Comms delivery failed |
