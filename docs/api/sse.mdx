MobKit uses Server-Sent Events (SSE) for two related real-time flows:

- interaction-scoped streaming through `POST /interactions/stream`
- event subscription and replay through `mobkit/events/subscribe`

## Interaction stream

```
POST /interactions/stream
```

Start an interaction with a specific member and receive an SSE response stream (`Content-Type: text/event-stream`).

### Request body

```json
{
  "member_id": "lead-1",
  "message": "Analyze the latest delivery failures."
}
```

### Frame format

```
id: evt-1
event: text_delta
data: {"type": "text_delta", "delta": "Analyzing the metrics now..."}

id: evt-2
event: tool_call
data: {"type": "tool_call", "name": "query_metrics", "args": {...}}

id: evt-3
event: interaction_complete
data: {"type": "interaction_complete", "result": "Done."}
```

## Event subscription

Use the `mobkit/events/subscribe` JSON-RPC method to request a point-in-time event snapshot plus replay frames.

Example request:

```json
{
  "jsonrpc": "2.0",
  "id": "sub-1",
  "method": "mobkit/events/subscribe",
  "params": {
    "scope": "mob",
    "last_event_id": "evt-42"
  }
}
```

The response includes:

| Field | Description |
|-------|-------------|
| `keep_alive.interval_ms` | Server keep-alive cadence |
| `keep_alive.event` | Keep-alive event name |
| `event_frames` | SSE-ready frames for replay |
| `events` | Parsed event payloads |

Supported scopes:

| Scope | Notes |
|-------|-------|
| `mob` | Whole runtime |
| `agent` | Requires `agent_id` |
| `interaction` | Interaction-scoped replay |

## Keep-alive

The server sends keep-alive frames at a configurable interval (default: 15 seconds) to prevent connection timeouts:

```
: keep-alive
```

## Sending messages

To send a message to an agent, use the `mobkit/send_message` RPC method and capture the returned `session_id` for correlation:

```json
{
  "accepted": true,
  "member_id": "lead-1",
  "session_id": "0195c3f2-8f40-7f95-a5a7-cc4f7d90d3d1"
}
```

Then observe streamed responses via `POST /interactions/stream` and broader replayable event history via `mobkit/events/subscribe`.

## Error handling

If the target member does not exist or the interaction request is malformed, the interaction endpoint returns an HTTP error response:

| Status | Cause |
|--------|-------|
| `400` | Empty or invalid member ID / malformed body |
| `404` | Member not found |
| `500` | Internal server error |
