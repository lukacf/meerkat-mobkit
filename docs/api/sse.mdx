MobKit uses Server-Sent Events (SSE) for real-time observation of agent and mob activity. Two endpoints provide per-agent and mob-wide event streams.

## Per-agent event stream

```
GET /agents/:agent_id/events
```

Subscribe to events from a single agent. The response is an SSE stream (`Content-Type: text/event-stream`).

### Frame format

```
id: lead-1:0
event: text_delta
data: {"type": "text_delta", "delta": "Analyzing the metrics now..."}

id: lead-1:1
event: tool_call
data: {"type": "tool_call", "name": "query_metrics", "args": {...}}

id: lead-1:2
event: interaction_complete
data: {"type": "interaction_complete", "result": "Done."}
```

## Mob-wide event stream

```
GET /mob/events
```

Subscribe to a merged stream of events from all agents in the mob.

### Frame format

```
id: mob:0
event: text_delta
data: {"agent_id": "lead-1", "event": {"type": "text_delta", "delta": "..."}}

id: mob:1
event: tool_call
data: {"agent_id": "worker-1", "event": {"type": "tool_call", "name": "search", "args": {}}}
```

Each frame has:

| Field | Description |
|-------|-------------|
| `id` | Unique frame identifier |
| `event` | Event type from the agent event model |
| `data` | JSON payload with agent attribution |

## Keep-alive

The server sends keep-alive frames at a configurable interval (default: 15 seconds) to prevent connection timeouts:

```
: keep-alive
```

## Sending messages

To send a message to an agent, use the `mobkit/send_message` RPC method (fire-and-forget) and observe responses via the SSE event streams above.

## Error handling

If the target agent does not exist, the SSE endpoint returns an HTTP error response:

| Status | Cause |
|--------|-------|
| `400` | Empty or invalid agent ID |
| `404` | Agent not found |
| `500` | Internal server error |
